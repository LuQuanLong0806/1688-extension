## 任务目标

对**项目所有源码**进行全量扫描、模块划分、功能梳理，生成**全覆盖、可集成CI/CD、高覆盖率**的单元测试套件；包含语法校验、依赖隔离、异常场景、覆盖率配置与报告。

## 触发句式

- 全项目生成完整单元测试
- 全覆盖单元测试 + 质量报告
- 为整个项目写单测，CI可用
- 全模块扫描并生成测试套件

## 执行流程（必须严格按顺序）

1. **全项目扫描与模块划分**
   - 扫描 `src/`、`lib/`、`main/` 等源码目录
   - 按**模块/层/功能**分组（如：api、service、utils、components）
   - 输出：模块清单 + 各模块文件数/函数数

2. **全局语法/格式/类型检查**
   - 执行项目全局校验：lint + 类型检查 + 编译
   - 有错误：列出并优先修复关键语法/类型问题
   - 确保：**无编译错误、无严重格式问题**再生成测试

3. **初始化测试环境（如缺失）**
   - 生成/补全配置：
     - JS/TS：`vitest.config.ts` / `jest.config.js`（含覆盖率阈值 ≥70%）
     - Python：`pytest.ini` + `requirements-test.txt`
     - Java：`pom.xml` / `build.gradle` 测试依赖
   - 创建全局 Setup：Mock 全局依赖、环境变量、公共 Fixture

4. **逐模块生成全覆盖测试**
   - 对每个模块的**所有公开函数/类/接口**生成测试
   - 覆盖标准：
     - ✅ 正常场景（所有主流程）
     - ✅ 边界值（数值、字符串、数组、对象）
     - ✅ 异常/错误（非法输入、依赖失败、超时）
     - ✅ 分支覆盖（if/else、switch、try/catch）
     - ✅ 依赖隔离（所有外部调用必须 Mock）
   - 命名/位置：严格遵循项目现有规范
