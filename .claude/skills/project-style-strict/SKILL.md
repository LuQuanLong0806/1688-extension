## 核心规则

1. **样式体系严格沿用**
   必须复用项目已有类名、CSS变量、间距、圆角、阴影、字体体系，不得更换字体、改动基础设计范式，不创造全新视觉风格

2. **配色使用规范**

- 主色、功能色、中性色优先直接取用项目既定色值变量
- 基础配色不足时，**仅在原有色相基础上调整明度、饱和度**衍生深浅变体，不新增陌生色系、不引入风格外色彩
- 所有颜色依旧绑定主题变量，明暗模式自动适配，杜绝固定色值硬编码

3. **主题切换强制兼容**
   背景、文字、边框、交互态样式全部依托项目主题变量编写，切换明暗模式无样式错乱、文字看不清问题

4. **交互与布局统一**
   hover动效、过渡时长、组件层级、排版逻辑和现有页面保持一致，新增模块视觉观感无缝衔接

5. **代码输出要求**
   代码格式贴合项目规范，不破坏原有结构，不引入冲突样式，产出可直接嵌入使用

6. **禁止行为**
   严禁脱离项目色调体系自创配色；禁止替换字体、修改基础圆角尺寸、变更整体布局风格
