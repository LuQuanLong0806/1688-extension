## 任务目标

仅对**本次变更（新增/修改）的文件**生成高质量、可直接运行的单元测试；必须包含语法校验、格式检查、链路覆盖、异常边界，并自动执行验证。

## 触发句式

- 为本次改动写单元测试
- 生成当前修改的单测并检查语法
- 增量单元测试 + 全链路覆盖
- 只测本次变更，跑通并修复

## 执行流程（必须严格按顺序）

1. **获取变更范围**
   - 列出本次新增/修改文件（不含删除）
   - 只处理源码文件（.ts/.js/.py/.java/.go 等），跳过配置/文档

2. **语法与格式检测（强制）**
   - JS/TS：执行 `eslint` + `tsc --noEmit`
   - Python：执行 `flake8` + `mypy`
   - Java：`mvn compile` / `gradle compileJava`
   - 有错误：先修复语法/格式，再继续；**禁止生成带语法错的测试**

3. **分析代码与链路**
   - 提取变更中的**所有公开函数/方法/接口/类**
   - 梳理调用链路：输入 → 处理 → 输出/副作用 → 依赖
   - 识别：正常场景、边界值、空值、非法输入、异常抛出

4. **生成单元测试（覆盖要求）**
   - 框架：自动沿用项目现有（Jest/Vitest/pytest/JUnit）
   - 位置：与源码对应（如 `src/foo.ts` → `src/__tests__/foo.test.ts`）
   - 用例必须包含：
     - ✅ 正常路径（主逻辑）
     - ✅ 边界条件（0、空、极值、特殊字符）
     - ✅ 异常/错误输入（非法类型、缺失参数）
     - ✅ 链路覆盖（关键调用、分支、返回值）
   - Mock：外部依赖（API、DB、缓存）必须 Mock，不连真实服务

5. **写入并运行测试**
   - 生成独立可运行测试文件，不改动源码
   - 执行项目测试命令（如 `npm test` / `pytest`）
   - 失败：自动修复断言、Mock、用例，直到通过
   - 输出：覆盖率摘要（行/分支/函数）

## 输出格式

1. 变更文件清单 + 语法/格式检查结果
2. 每个文件的测试代码（完整可复制）
3. 运行命令 + 执行结果 + 覆盖率
4. 如有失败：问题说明 + 修复方案

## 约束

- ❌ 不扫描全项目，**只处理本次变更**
- ❌ 不生成未使用用例、不写冗余注释
- ✅ 测试必须**可直接运行且通过**
