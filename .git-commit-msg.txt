阶段十：扩展自动登录审核文档复审 + 真实问题修复

依据 docs/extension-auto-auth-audit-2026-06-13.md 复审，结合实际场景：

【文档核对】Step 1-7 全部属实，无虚构
【异议评估】4 项可优化点里 2 项描述不准、2 项无异议

【修复 A】collect-data.js + dxm-config.js fetch 前等 token
- 根因：启动时 autoGetToken 异步，用户立即操作时 localStorage 可能没 token
  → fetch 不带 Authorization → 静默 401
- 新增 ensureToken (collect-data) / withAuth (dxm-config) helper
- saveToServer / checkExists 包一层 ensureToken

【修复 B】checkExists 区分错误类型
- 之前 catch 无差别 callback({exists:false})，让上层无法区分
  "真不存在" vs "鉴权失败" vs "网络异常"
- 改为 callback({exists:false, error:'auth'|'network'})
- float-btn.js 处理 auth/network 错误显式提示

【修复 C】服务端 cookie 行为单元测试（防回归）
- 现有 login/plugin-auth 测试完全不验证 res.cookie
- 新建 cookie-auth.test.js 7 个测试：
  login/plugin-login/change-password/logout 的 Set-Cookie
  HttpOnly/SameSite=Lax/Max-Age=604800 属性
  cookie.value === body.token 一致性
